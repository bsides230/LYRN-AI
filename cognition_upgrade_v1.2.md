# LYRN-AI Elements Spec v1

## Global Context
- **Build Prompt Order:** `build_prompt → chat_pair → delta → [repeat] → system_flags`
- **KV Cache Rule:** No reloads; new context is appended only.
- **Status Files:** Write small JSON snapshots for GUI to read (health, last_run, counts).
- **Markers/Flags:** Prefer triple-`#` sentinels for jobs and blocks.
- **End Marker Style:** `###<NAME>_START### … ###_END###`

---

## Phase 1: Core

### E01 — Delta Manager
- **Goal:** To manage live, non-destructive updates to the system's memory context (Snapshot) without reloading the entire prompt, thus preserving the KV cache.
- **Responsibilities:** Create, store, and manifest delta files.
- **Outputs:** Timestamped delta files (`/deltas/YYYY/MM/DD/<delta_id>.txt`) and an updated `_manifest.json`.

### E02 — Snapshot Loader
- **Goal:** To load the foundational static and dynamic context into memory at the start of a session, forming the base for the KV cache.
- **What exists today:** The `PromptBuilder` class in `lyrn_gui_v6.8.pyw`.

### E03 — Build-Prompt Structure
- **Goal:** Keep a single stable prompt while appending live context.
- **Section Order:** `build_prompt → chat_pair → delta → [repeat] → system_flags`

### E04 — System Flags & Job Format
- **Goal:** Simple triggerable jobs with zero code branching in the prompt.
- **Standard Job Header:** `###JOB_START: <JOB_NAME>### ... ###_END###`

### E05 — Heartbeat Cycle
- **Goal:** Act as the LLM's "internal dialog." It serves as a second LLM pass for summaries, self-correction, output alignment checks, and triggering actions via affordances.
- **Trigger:** The Automation Controller runs this job after every reasoning cycle.
- **Output Template:** Uses `###HB_..._START###` and `###_END###` markers for structured output (summaries, deltas, actions).

### E06 — Core Integration
- **Goal:** Wire Delta ↔ Snapshot ↔ Heartbeat with a simple, internal application loop.
- **Crash Safety:** Use tmp-write → fsync → rename pattern for all file writes.

### E07 — Personality Sliders & Presets
- **Goal:** Live 0–1000 trait tuning via `personality.json` with delta writes.
- **UI:** A popup with sliders and save/load preset functionality.

### E08 — Timing & Handoffs
- **Goal:** Finish-to-finish sequencing; no concurrent writes.
- **Mechanism:** A single, system-wide file lock (`lyrn.lock`).

### E09 — Automation Controller
- **Goal:** Standardize job definitions and scheduling. This is a dependency for the Heartbeat Cycle.
- **Scheduler Rules:** Run jobs during idle windows or based on resource thresholds.

---

## Phase 2: Episodic & Relational Memory

### E10 — Verbatim Chat Entry Format
- **Goal:** Plain-text, human-auditable chat memory using a structured `/entry ... /end_entry` format.
- **File Location:** `/chat/YYYY/MM/DD/<entry_id>.txt`

### E11 — Chat Pair Index & Browser
- **Goal:** A scrollable search UI (popup window) to browse and select chat history.
- **Actions:** Multiselect to add to `chat_review.txt`; right-click to quote to `quote.txt`.

### E12 — Topic Indexing Engine
- **Goal:** Nuanced long-term memory with simple search and bulk-activate.
- **UI:** A popup with tabs for viewing/editing topic indexes.

---

## Phase 3: Advanced Cognition

### E13 — Reflection Job
- **Goal:** Periodic consolidation of memory, resolving contradictions, and proposing next-steps.
- **Trigger:** Time-based or after N chat pairs.

### E14 — Dream Sequence Job
- **Goal:** Idle-time creativity and insight generation with lower confidence.
- **Trigger:** System idle for ≥ X seconds.

### E15 — Internal Goals System
- **Goal:** Enable the AI to set, track, and pursue long-term, autonomous goals.
- **Storage:** A `goals.json` file.
- **Execution:** Active goals are loaded into the prompt to guide the AI's actions during idle time.

---

## External Adapters

### E16 — TTS, STT, and Web Tool
- **Goal:** Provide voice I/O and the ability to search the web for live information.
- **Trigger:** Action blocks (`###ACTION_START###...`) generated by the Heartbeat Cycle.

### E17 — System Action Toolkit
- **Goal:** Allow the AI to perform safe, predefined actions on the local OS.
- **Security:** All actions are validated against a strict allowlist. Sensitive actions require user confirmation via a GUI popup.

### E18 — Calendar, Contacts, and Notifications Adapters
- **Goal:** Extend the AI's capabilities to be a true personal assistant by managing schedules, contacts, and sending reminders.

### E19 — Affordance Creator
- **Goal:** A user-friendly GUI ("popup template builder") to define new tools and actions for the AI without writing any code.

## Parser Contract (v1.0)

### Global rules
- **Markers:** Every block begins with `###<BLOCK>_START###` and ends with `###_END###` (exact, uppercase).
- **Enable switch:** The first logical field in each block MAY be `ENABLED=true|false`. If `false`, parsers MUST ignore the remainder of that block until `###_END###`.
- **Key:Value lines:** Fields are `KEY: value` on single lines. Booleans are `true|false`. Integers are base‑10. Times are ISO‑8601 UTC (`YYYY‑MM‑DDTHH:MM:SSZ`).
- **Arrays:** When present inline, arrays MUST be JSON (e.g., `["alpha","beta"]`), not comma lists.
- **Heredoc values:** Use `<<EOF`; the value is the following lines until a lone `EOF` on its own line.
- **No tabs:** Parsers MUST treat TAB as invalid; use spaces only.
- **Whitespace:** Parsers MUST trim trailing spaces but preserve interior whitespace of heredocs.
- **Order:** Block order is stable but not required by parsers. Within a block, field order is not significant.

### Blocks

#### 1) `###HB_META_START### … ###_END###`
Required fields:
- `SCHEMA: "hb/v1"`
- `VERSION: "1.0.x"`
- `HB_ID: "hb_YYYY-MM-DDTHH-MM-SSZ_xxxx"`
- `CHAT_PAIR_ID: "cp_YYYY-MM-DDTHH-MM-SSZ"`
- `TIMESTAMP_UTC: "YYYY-MM-DDTHH:MM:SSZ"`
- `PROCESSING_MODE: "speaking"|"thinking"`
- `REASONING_FLAG: "none"|"needs_follow_up"|"needs_search"|"needs_action"`
Optional fields:
- `EPISODE_ID: "ep_..."`
- `MODEL_NAME: "..."`
- `KV_CACHE_KEEP_PAIRS: <int>`
- `NOTES: <<EOF … EOF`

_Example:_
```
###HB_META_START###
ENABLED=true
SCHEMA: "hb/v1"
VERSION: "1.0.0"
HB_ID: "hb_2025-08-09T11-10-00Z_0001"
CHAT_PAIR_ID: "cp_2025-08-09T11-09-42Z"
TIMESTAMP_UTC: "2025-08-09T11:10:00Z"
PROCESSING_MODE: "thinking"
REASONING_FLAG: "needs_follow_up"
MODEL_NAME: "gemma-4b-q4"
KV_CACHE_KEEP_PAIRS: 3
###_END###
```

#### 2) `###HB_SUM_START### … ###_END###`
- Contains a concise, factual summary of the current chat pair.
- Either single-line `TEXT: ...` or heredoc `TEXT: <<EOF ... EOF`.

_Example:_
```
###HB_SUM_START###
ENABLED=true
TEXT: <<EOF
User asked to rewire the GUI prompt loader; model responded with a plan and
a diff summary. No changes applied yet.
EOF
###_END###
```

#### 3) `###HB_KEYS_START### … ###_END###`
- `KEYWORDS: ["k1","k2",...]`
- `TOPICS: ["t1","t2",...]`
- `ENTITIES: [ ["name","type"], ... ]`  // type examples: "person","org","file","path","script"

_Example:_
```
###HB_KEYS_START###
ENABLED=true
KEYWORDS: ["delta","manifest","kv cache"]
TOPICS: ["prompt building","heartbeat"]
ENTITIES: [["LYRN","org"],["lyrn_gui.py","file"]]
###_END###
```

#### 4) `###HB_INSIGHTS_START### … ###_END###`
- `NOTES: <<EOF … EOF` (bulleted or paragraph). May include `inferred:` tags explicitly.

_Example:_
```
###HB_INSIGHTS_START###
ENABLED=true
NOTES: <<EOF
- Consolidate manifest writes after fsync rename.
- inferred: GUI may drop to 10fps under high IO.
EOF
###_END###
```

#### 5) `###HB_MEMORY_START### … ###_END###`  (Structured delta lines)
Each delta line grammar:
```
DELTA|<scope>|<target>|<op>|<path>|<value_mode>|<value or EOF>
```
- `scope`: `conversation|profile|world|goals|flags`
- `target`: freeform leaf (e.g., `conversation_summary`,`keywords`,`system_flags`)
- `op`: `append|set|upsert|remove`
- `path`: dot‑notation into the Live Snapshot (e.g., `conversation.summary`)
- `value_mode`: `RAW|EOF` (if `EOF`, the very next line begins the value and ends at a lone `EOF`)
- `value`: if `RAW`, the remainder of the line; if `EOF`, multi‑line body.

_Example (mixed values):_
```
###HB_MEMORY_START###
ENABLED=true
DELTA|conversation|conversation_summary|append|conversation.summary|EOF|
This pair adds a note about manifest integrity.
EOF
DELTA|flags|system_flags|set|flags.next_job_ready|RAW|true
###_END###
```

#### 6) `###HB_ACTIONS_START### … ###_END###`  (Declarative intents only)
Two allowed intent lines:
- `AFFORD|<name>|<args_json>`
- `JOB|<name>|<args_json>`

_Example:_
```
###HB_ACTIONS_START###
ENABLED=true
AFFORD|open_file|{"path":"automation/queued_chunks/chunk_001.txt"}
JOB|enqueue|{"name":"summary_job","priority":2}
###_END###
```

#### 7) `###HB_FOLLOWUPS_START### … ###_END###`
Prompts or corrections to pursue later:
- `ASK|<question_id>|<prompt>`
- `CORRECT|<field>|<correction_text>`

_Example:_
```
###HB_FOLLOWUPS_START###
ENABLED=true
ASK|q_0001|Please confirm the intended KV cache keep_pairs value.
CORRECT|doc/path|The job list uses JOB_START tags, not SECTION tags.
###_END###
```

#### 8) `###HB_SEARCH_START### … ###_END###`
- `SEARCH|<query>|<hints_json>`

_Example:_
```
###HB_SEARCH_START###
ENABLED=false
SEARCH|customtkinter CTkFont error|{"priority":1,"timebox_min":5}
###_END###
```

#### 9) `###HB_FLAGS_START### … ###_END###`
- `NEXT_ACTION: "idle"|"run_jobs"|"wait_gui"|"search"|"affordance"`
- `SAFE_AFTER_DELETE_SIGNAL: true|false`

_Example:_
```
###HB_FLAGS_START###
ENABLED=true
NEXT_ACTION: "run_jobs"
SAFE_AFTER_DELETE_SIGNAL: true
###_END###
```

#### 10) `###HB_METRICS_START### … ###_END###` (optional)
- `EVAL_TOKS_PER_SEC: <float>`
- `GEN_TOKS_PER_SEC: <float>`
- `LAT_MS: <int>`
- `HB_MS: <int>`

_Example:_
```
###HB_METRICS_START###
ENABLED=true
EVAL_TOKS_PER_SEC: 130.4
GEN_TOKS_PER_SEC: 28.9
LAT_MS: 420
HB_MS: 165
###_END###
```

### Validation checklist
- All required META fields present.
- No tabs; heredocs balanced; JSON parses.
- Unknown ops or enums MUST hard‑fail with a line/col reference.
- If any block `ENABLED=false`, parser MUST skip its content without error.
- Outputs: (a) `hb.json` (normalized structured form), (b) `deltas/*.json` created from MEMORY lines, (c) `system_flags.json` patch if applicable.
